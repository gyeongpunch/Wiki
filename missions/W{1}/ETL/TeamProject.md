## 만약 데이터가 갱신되면 과거의 데이터는 어떻게 되어야 할까요? 과거의 데이터를 조회하는 게 필요하다면 ETL 프로세스를 어떻게 변경해야 할까요?

### 1. DB를 백업하여 새로운 DB file을 만든다.
    * 6개월마다 한 번씩 update하는 과정이므로 연 단위로 보면 overhead가 크지 않을 것으로 예상된다.
    * 그러므로 미리 정해둔 주기마다 새로운 DB file을 만들어 과거의 Data를 확인할 수 있게 하였다. 

### 2. DB는 그대로 두고 update colulmn을 추가하고 행을 INSERT한다.
    * DB file은 추가하지 않고 update하는 날짜의 column을 추가한다.
    * 또한 정해둔 주기마다 Data를 불러와서 새로운 행에 저장하는 방식이 있다.
    * 1번보다 overhead는 작을 것으로 예상된다. 또한 다른 DBMS에 적용할 때 비교적 용이할 것으로 예상된다.

# ETL process를 왜 나누었을까?

## 1. Extract
### 목적
* 데이터 수집: 다양한 출처에서 데이터를 추출하여 하나의 통합된 데이터 세트로 만드는 것이 목적이다. 출처는 관계형 데이터베이스, NoSQL 데이터베이스, 파일 시스템, API, 웹 서비스 등 다양하다.
* 데이터 가용성 보장: 필요한 데이터를 중앙 저장소에 모아서 비즈니스 분석, 보고, 데이터 과학 및 머신 러닝 모델링 등에 사용 가능하도록 한다.

### 이유
* 데이터 통합: 여러 시스템에 흩어져 있는 데이터를 한 곳에 모아야 전체적인 비즈니스 상황을 파악할 수 있다. 통합된 데이터는 일관성 있는 보고 및 분석을 가능하게 한다.
* 데이터 품질 개선: 다양한 출처에서 데이터를 추출하여 품질 검사를 통해 데이터의 정확성과 일관성을 확보할 수 있다. 추출 과정에서 데이터의 정합성, 중복 여부 등을 확인하고 문제를 조기에 발견할 수 있다.
* 시간 절약: 분석과 보고에 필요한 데이터를 매번 수동으로 수집하는 대신 자동화된 ETL 프로세스를 통해 효율성을 높이고 시간을 절약할 수 있다.
* 업데이트 및 동기화: 운영 데이터베이스와 분석 데이터베이스 간의 동기화를 통해 최신 데이터를 반영할 수 있다. 실시간 혹은 주기적으로 데이터를 추출하여 최신 상태를 유지한다.
* 데이터 액세스: 다양한 출처에서 데이터를 추출하여 데이터 소비자가 쉽게 접근할 수 있도록 한다. 이는 비즈니스 인사이트를 얻고 의사결정을 내리는 데 중요한 역할을 한다.

## 2. Transfrom
### 목적
* 데이터 정제: 추출한 데이터를 일관성 있고 신뢰할 수 있게 만들기 위해 데이터 오류를 수정하고 불필요한 데이터를 제거한다.
* 데이터 구조 변경: 데이터를 분석 및 보고에 적합한 형식으로 변환한다. 이는 데이터 모델링 및 데이터 웨어하우스 설계에 맞춰 데이터를 변형하는 작업을 포함한다.
* 데이터 통합: 다양한 소스에서 가져온 데이터를 하나의 일관된 형식으로 통합하여 사용자가 쉽게 접근하고 이해할 수 있도록 한다.
* 비즈니스 규칙 적용: 비즈니스 로직과 규칙을 적용하여 데이터를 처리한다. 이는 계산, 집계, 필터링, 데이터 매핑 등의 작업을 포함한다.
### 이유
* 데이터 품질 향상: 데이터를 정제하고 일관성을 확보하여 분석 및 보고에 신뢰할 수 있는 데이터를 제공한다. 이는 데이터 정확성과 완전성을 높여준다.
* 일관성 있는 데이터 제공: 다양한 소스에서 추출한 데이터는 형식과 구조가 다를 수 있다. 변환 과정을 통해 데이터를 표준화하여 일관성 있는 데이터를 제공한다.
* 효율적인 분석 가능: 데이터를 분석 목적에 맞게 구조화하고 집계하여 분석 작업을 보다 효율적으로 수행할 수 있다. 이는 데이터 웨어하우스나 데이터 레이크에서의 쿼리 성능을 향상시킨다.
* 비즈니스 요구 사항 충족: 비즈니스 규칙과 요구 사항에 맞춰 데이터를 변환하여 특정 비즈니스 인사이트를 도출할 수 있도록 한다. 예를 들어, 매출 데이터를 월별로 집계하거나 고객 데이터를 세그먼트화할 수 있다.
* 데이터 통합: 서로 다른 시스템에서 추출한 데이터를 통합하여 전체적인 비즈니스 상황을 파악할 수 있게 한다. 이는 데이터 사이로 존재하는 사일로를 제거하고 전체적인 데이터 가시성을 향상시킨다.
* 데이터 보강: 외부 데이터를 추가하거나 계산된 데이터를 생성하여 기존 데이터를 보강한다. 예를 들어, 위치 데이터를 사용하여 지리적 분석을 수행하거나 고객 데이터를 사용하여 고객 세그먼트를 생성할 수 있다.

## 3. Load
### 목적
* 데이터 저장: 변환된 데이터를 데이터 웨어하우스, 데이터 마트, 데이터 레이크 등 목적지 시스템에 저장하여 사용자가 쉽게 접근하고 활용할 수 있도록 한다.
* 분석 준비: 데이터 분석, 비즈니스 인텔리전스(BI) 도구, 보고 및 데이터 과학 애플리케이션에서 데이터를 활용할 수 있도록 준비한다.
* 성능 최적화: 데이터베이스의 성능을 최적화하여 빠르고 효율적인 데이터 조회와 분석을 가능하게 한다.
### 이유
* 데이터 접근성 향상: 데이터를 한 곳에 중앙화하여 조직 내 다양한 부서와 사용자가 쉽게 접근하고 활용할 수 있도록 한다. 이는 데이터 접근성을 높여 의사결정 속도를 향상시킨다.
* 일관성 유지: 모든 사용자가 동일한 데이터를 사용하도록 하여 데이터의 일관성과 신뢰성을 유지한다. 이는 조직 내에서 공통된 데이터 소스를 사용하게 하여 데이터의 정확성을 보장한다.
* 실시간 또는 배치 처리: 데이터 로드 과정은 실시간으로 또는 배치(batch)로 수행될 수 있다. 이는 비즈니스 요구사항에 맞춰 최신 데이터를 제공하거나 대량의 데이터를 효율적으로 처리할 수 있게 한다.
* 데이터 통합: 다양한 소스에서 수집한 데이터를 통합하여 전체적인 비즈니스 상황을 파악할 수 있게 한다. 이는 데이터 웨어하우스에 데이터를 로드하여 이루어지며, 데이터 분석과 보고서 작성을 용이하게 한다.
* 성능 향상: 데이터 웨어하우스나 데이터 레이크에 데이터를 로드함으로써, 분석 쿼리와 보고서 생성 시 더 빠른 응답 시간을 제공한다. 이는 최적화된 데이터 저장 구조를 통해 가능하다.
* 데이터 보관: 장기적으로 데이터를 저장하고 보관하여 히스토리컬 데이터 분석을 가능하게 한다. 이는 과거 데이터를 분석하여 트렌드를 파악하거나 예측 모델을 구축하는 데 유용하다.
* 데이터 가용성 보장: 데이터 로드를 통해 변환된 데이터를 신속하게 사용할 수 있게 하여 비즈니스 인텔리전스 도구, 데이터 분석 애플리케이션 및 기타 시스템에서 즉시 활용할 수 있도록 한다.
