# Jupyter는 어떤 툴이고 언제 어떻게 사용할 수 있는지를 설명하세요.
## Jupyter Notebook
* 데이터 과학자, 연구자, 교육자, 개발자 등이 다양한 프로그래밍 언어로 작성한 코드를 실행하고, 데이터를 분석하며, 시각화 결과를 공유할 수 있는 웹 기반의 인터랙티브 환경이다. 이 도구는 주로 Python을 포함한 여러 언어를 지원하며, 코드와 텍스트, 시각화 결과를 한 곳에서 통합적으로 관리할 수 있는 기능을 제공한다.

### 주요 특징
* 인터랙티브 코딩: 코드 셀에서 작성한 코드를 즉시 실행하고 결과를 확인할 수 있습니다. 이는 반복적 실험과 디버깅에 매우 유용하다.
* 문서1화와 시각화: 마크다운(Markdown) 셀을 통해 설명, 수식, 링크 등을 추가할 수 있으며, 다양한 라이브러리를 사용해 데이터를 시각화할 수 있다.
* 다양한 언어 지원: Jupyter는 주로 Python을 사용하지만, 커널을 통해 R, Julia, SQL 등 여러 언어를 지원다.
* 오픈 소스: 누구나 자유롭게 사용할 수 있으며, 커뮤니티가 활발하게 발전시키고 있다.

### 사용 시기 및 방법
* 데이터 분석과 시각화: 데이터 세트를 탐색하고 시각화할 때, Jupyter Notebook은 매우 유용합니다. 실시간으로 코드를 실행하고, 결과를 바로 확인할 수 있다.
* 기계 학습 및 데이터 과학 프로젝트: 모델을 훈련시키고, 평가하며, 성능을 시각화하는 과정에서 단계별로 진행 상황을 문서화할 수 있다.
* 교육과 학습: 코드를 작성하고 실행하는 과정을 단계별로 설명할 수 있어, 프로그래밍 및 데이터 과학 교육에 유용하다.
* 프로토타이핑: 아이디어를 빠르게 구현하고 테스트할 때, Jupyter Notebook을 사용하면 효과적이다.
* 방법: Anaconda를 통해 설치할 수 있다. Jupyter환경이 설치되면 Jupyter Notebook을 실행할 수 있으며, 이은 데이터 분석, 시각화, 기계 학습 등 다양한 용도로 활용될 수 있는 매우 유용한 도구이다.

# pandas 라이브러리는 왜 만들어졌고 어디에 사용할 수 있는지 설명하세요.
## Pandas
* 데이터 조작 및 분석을 위해 개발된 Python 라이브러리이다. Pandas는 특히 테이블과 같이 구조화된 데이터를 다루는 데 강력한 기능을 제공하며, 데이터 프레임(DataFrame)을 중심으로 다양한 기능을 제공한다.
### Pandas가 만들어진 이유
* 데이터 처리 효율성: 데이터 분석 작업에서 대용량 데이터를 효율적으로 처리하고 조작할 수 있는 도구의 필요성에 따라 개발되었다. Pandas는 이를 위해 고성능의 데이터 구조와 연산 기능을 제공한다.
* 쉬운 데이터 조작: 복잡한 데이터 조작을 간단하고 직관적인 코드로 수행할 수 있게 한다. 이는 데이터 필터링, 정렬, 집계, 변환 등의 작업을 보다 쉽게 할 수 있도록 도와준다.
* Python의 확장성: Python은 데이터 과학 및 분석에서 널리 사용되는 언어이며, Pandas는 Python의 생태계와 잘 통합되어 다른 라이브러리(예: NumPy, Matplotlib, Scikit-learn 등)와 함께 사용하기 좋다.
* 데이터 분석의 편의성: Pandas는 데이터 분석을 위한 다양한 기능을 제공하여 데이터의 탐색, 정제, 전처리, 시각화 등을 쉽게 수행할 수 있게 한다.
### 사용 사례
* 데이터 탐색: 데이터 세트의 기본 특성을 이해하고, 결측값이나 이상치를 탐색하는 데 사용된다.
* 데이터 전처리: 모델링을 위한 데이터 클렌징, 변환, feature engineering 등을 수행한다.
* 데이터 분석: 특정 비즈니스 문제를 해결하기 위한 데이터 분석 및 통계 작업에 사용된다.
* 리포팅 및 시각화: 분석 결과를 도식화하여 보고서를 작성하거나 데이터를 시각적으로 표현하는 데 유용하다.
* ETL 작업: Data Extract, Transform, Load ETL 파이프라인의 일부로서 데이터의 정제 및 변환 작업을 자동화하는 데 사용된다.

# ETL 프로세스는 왜 만들어졌고 왜 3단계로 작업을 하는지에 대해 설명하세요.
## ETL 프로세스가 만들어진 이유
* 데이터 통합: 기업은 여러 출처에서 데이터를 수집한다. 예를 들어, ERP 시스템, CRM 시스템, 로그 파일, 소셜 미디어 데이터 등이 있다. ETL은 이러한 다양한 소스의 데이터를 하나의 중앙 시스템으로 통합한다.
* 데이터 품질 향상: 다양한 소스에서 수집된 데이터는 일관성, 정합성, 품질 등의 문제가 발생할 수 있다. ETL 프로세스를 통해 데이터를 정제하고 변환하여 품질을 향상시킬 수 있다.
* 효율적 데이터 분석: 통합되고 정제된 데이터는 분석, 보고, 비즈니스 인텔리전스(BI) 작업에 적합하다. ETL은 데이터 웨어하우스 같은 분석 시스템으로 데이터를 준비한다.
* 업무 효율성 증대: 데이터 통합과 정제를 자동화함으로써 데이터 분석과 관련된 수작업을 줄이고, 이를 통해 업무 효율성을 높인다.
* 실시간 또는 배치 처리: 실시간 또는 주기적으로 데이터를 처리하여 최신 상태의 데이터를 유지할 수 있다.
## 1. Extract
### 목적
* 데이터 수집: 다양한 출처에서 데이터를 추출하여 하나의 통합된 데이터 세트로 만드는 것이 목적이다. 출처는 관계형 데이터베이스, NoSQL 데이터베이스, 파일 시스템, API, 웹 서비스 등 다양하다.
* 데이터 가용성 보장: 필요한 데이터를 중앙 저장소에 모아서 비즈니스 분석, 보고, 데이터 과학 및 머신 러닝 모델링 등에 사용 가능하도록 한다.

### 이유
* 데이터 통합: 여러 시스템에 흩어져 있는 데이터를 한 곳에 모아야 전체적인 비즈니스 상황을 파악할 수 있다. 통합된 데이터는 일관성 있는 보고 및 분석을 가능하게 한다.
* 데이터 품질 개선: 다양한 출처에서 데이터를 추출하여 품질 검사를 통해 데이터의 정확성과 일관성을 확보할 수 있다. 추출 과정에서 데이터의 정합성, 중복 여부 등을 확인하고 문제를 조기에 발견할 수 있다.
* 시간 절약: 분석과 보고에 필요한 데이터를 매번 수동으로 수집하는 대신 자동화된 ETL 프로세스를 통해 효율성을 높이고 시간을 절약할 수 있다.
* 업데이트 및 동기화: 운영 데이터베이스와 분석 데이터베이스 간의 동기화를 통해 최신 데이터를 반영할 수 있다. 실시간 혹은 주기적으로 데이터를 추출하여 최신 상태를 유지한다.
* 데이터 액세스: 다양한 출처에서 데이터를 추출하여 데이터 소비자가 쉽게 접근할 수 있도록 한다. 이는 비즈니스 인사이트를 얻고 의사결정을 내리는 데 중요한 역할을 한다.

## 2. Transfrom
### 목적
* 데이터 정제: 추출한 데이터를 일관성 있고 신뢰할 수 있게 만들기 위해 데이터 오류를 수정하고 불필요한 데이터를 제거한다.
* 데이터 구조 변경: 데이터를 분석 및 보고에 적합한 형식으로 변환한다. 이는 데이터 모델링 및 데이터 웨어하우스 설계에 맞춰 데이터를 변형하는 작업을 포함한다.
* 데이터 통합: 다양한 소스에서 가져온 데이터를 하나의 일관된 형식으로 통합하여 사용자가 쉽게 접근하고 이해할 수 있도록 한다.
* 비즈니스 규칙 적용: 비즈니스 로직과 규칙을 적용하여 데이터를 처리한다. 이는 계산, 집계, 필터링, 데이터 매핑 등의 작업을 포함한다.
### 이유
* 데이터 품질 향상: 데이터를 정제하고 일관성을 확보하여 분석 및 보고에 신뢰할 수 있는 데이터를 제공한다. 이는 데이터 정확성과 완전성을 높여준다.
* 일관성 있는 데이터 제공: 다양한 소스에서 추출한 데이터는 형식과 구조가 다를 수 있다. 변환 과정을 통해 데이터를 표준화하여 일관성 있는 데이터를 제공한다.
* 효율적인 분석 가능: 데이터를 분석 목적에 맞게 구조화하고 집계하여 분석 작업을 보다 효율적으로 수행할 수 있다. 이는 데이터 웨어하우스나 데이터 레이크에서의 쿼리 성능을 향상시킨다.
* 비즈니스 요구 사항 충족: 비즈니스 규칙과 요구 사항에 맞춰 데이터를 변환하여 특정 비즈니스 인사이트를 도출할 수 있도록 한다. 예를 들어, 매출 데이터를 월별로 집계하거나 고객 데이터를 세그먼트화할 수 있다.
* 데이터 통합: 서로 다른 시스템에서 추출한 데이터를 통합하여 전체적인 비즈니스 상황을 파악할 수 있게 한다. 이는 데이터 사이로 존재하는 사일로를 제거하고 전체적인 데이터 가시성을 향상시킨다.
* 데이터 보강: 외부 데이터를 추가하거나 계산된 데이터를 생성하여 기존 데이터를 보강한다. 예를 들어, 위치 데이터를 사용하여 지리적 분석을 수행하거나 고객 데이터를 사용하여 고객 세그먼트를 생성할 수 있다.

## 3. Load
### 목적
* 데이터 저장: 변환된 데이터를 데이터 웨어하우스, 데이터 마트, 데이터 레이크 등 목적지 시스템에 저장하여 사용자가 쉽게 접근하고 활용할 수 있도록 한다.
* 분석 준비: 데이터 분석, 비즈니스 인텔리전스(BI) 도구, 보고 및 데이터 과학 애플리케이션에서 데이터를 활용할 수 있도록 준비한다.
* 성능 최적화: 데이터베이스의 성능을 최적화하여 빠르고 효율적인 데이터 조회와 분석을 가능하게 한다.
### 이유
* 데이터 접근성 향상: 데이터를 한 곳에 중앙화하여 조직 내 다양한 부서와 사용자가 쉽게 접근하고 활용할 수 있도록 한다. 이는 데이터 접근성을 높여 의사결정 속도를 향상시킨다.
* 일관성 유지: 모든 사용자가 동일한 데이터를 사용하도록 하여 데이터의 일관성과 신뢰성을 유지한다. 이는 조직 내에서 공통된 데이터 소스를 사용하게 하여 데이터의 정확성을 보장한다.
* 실시간 또는 배치 처리: 데이터 로드 과정은 실시간으로 또는 배치(batch)로 수행될 수 있다. 이는 비즈니스 요구사항에 맞춰 최신 데이터를 제공하거나 대량의 데이터를 효율적으로 처리할 수 있게 한다.
* 데이터 통합: 다양한 소스에서 수집한 데이터를 통합하여 전체적인 비즈니스 상황을 파악할 수 있게 한다. 이는 데이터 웨어하우스에 데이터를 로드하여 이루어지며, 데이터 분석과 보고서 작성을 용이하게 한다.
* 성능 향상: 데이터 웨어하우스나 데이터 레이크에 데이터를 로드함으로써, 분석 쿼리와 보고서 생성 시 더 빠른 응답 시간을 제공한다. 이는 최적화된 데이터 저장 구조를 통해 가능하다.
* 데이터 보관: 장기적으로 데이터를 저장하고 보관하여 히스토리컬 데이터 분석을 가능하게 한다. 이는 과거 데이터를 분석하여 트렌드를 파악하거나 예측 모델을 구축하는 데 유용하다.
* 데이터 가용성 보장: 데이터 로드를 통해 변환된 데이터를 신속하게 사용할 수 있게 하여 비즈니스 인텔리전스 도구, 데이터 분석 애플리케이션 및 기타 시스템에서 즉시 활용할 수 있도록 한다.

# SQL은 왜 만들어졌고 어디에 사용하는지 설명하세요.
## SQL
* SQL(Structured Query Language)은 RDBMS에서 데이터를 관리하고 조작하기 위해 개발된 표준 프로그래밍 언어이다. SQL은 데이터베이스에서 데이터를 삽입, 조회, 업데이트, 삭제하는 기능을 제공한다.
### SQL이 만들어진 이유
* 데이터 관리 표준화: 1970년대에 IBM의 연구원들이 관계형 데이터 모델을 기반으로 데이터를 관리할 수 있는 표준 언어로 SQL을 개발했다. 이는 데이터베이스와 상호작용하는 표준화된 방법을 제공하여 일관성과 호환성을 보장한다.
* 데이터 조작의 용이성: SQL은 사용자가 직관적으로 데이터베이스와 상호작용할 수 있도록 설계되었다. 복잡한 데이터 구조를 간단한 명령어로 조작할 수 있다.
* 관계형 데이터 모델 지원: 관계형 데이터 모델은 데이터를 테이블 형식으로 저장하며, SQL은 이러한 구조를 효과적으로 관리하고 질의할 수 있는 기능을 제공한다.
* 데이터 무결성 및 보안: SQL은 트랜잭션 관리, 데이터 무결성 제약 조건, 사용자 권한 관리 등을 통해 데이터의 정확성과 보안을 유지한다.
### SQL 주요 용도
* DDL(Data Definition Language)
    * 테이블 생성 및 수정: 데이터베이스 객체(테이블, 인덱스, 뷰 등)를 정의하고 수정하는 데 사용된다.
* DML(Data Manipulation Language)
    * 데이터 삽입, 업데이트, 삭제, 조회: 테이블 내 데이터를 조작하는 데 사용된다.
* DCL(Data Control Language)
    * 권한 부여 및 회수: 데이터베이스 사용자 권한을 관리하는 데 사용된다.
* TCL(Transaction Control Language)
    * 트랜잭션 관리: 데이터베이스 트랜잭션을 시작, 커밋, 롤백하는 데 사용됩니다.
